## Goal
Capture `usage` and `timings` from API responses to understand why vLLM is 10x slower than llama-server.

## Changes needed:

### 1. `bench/client.py`
- Return a richer response object: `(content, usage_dict)` instead of just `str`
- `usage_dict` includes: `prompt_tokens`, `completion_tokens`, and `timings` (if present from llama-server)
- Update the preflight caller too

### 2. `bench/runner.py`
- Unpack the new return value from `chat_complete`
- Store token counts and timing data in `_Run`
- Dump the new fields to JSON
- Display throughput info in CLI output (tok/s generation speed)

### 3. `analysis/visualize.py`
- Add throughput data to existing charts (hover text)
- Enhance `speed_vs_quality` chart to show throughput labels
- Maybe add a dedicated throughput comparison chart

## Data available:
- **Both servers**: `usage.prompt_tokens`, `usage.completion_tokens`
- **llama-server only**: `timings.prompt_n`, `timings.prompt_ms`, `timings.prompt_per_second`, `timings.predicted_n`, `timings.predicted_ms`, `timings.predicted_per_second`, `timings.cache_n`
- **vLLM**: only token counts (no server-side timing breakdown)
