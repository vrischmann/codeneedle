Two related fixes:

### 1. Debug request/response logging (`--debug` flag)
- When `--debug` is passed, dump the full request payload and raw JSON response for every query
- Store in `results/<corpus>__<model>__debug.json` alongside the normal dump
- Also log tokenized prompt length and response structure

### 2. Fix `enable_thinking` for vLLM chat template
The NVFP4 chat template emits `<think>\n` (open) when `enable_thinking=true` (default). Combined with `prefill_no_think`, this creates a double assistant turn and causes the model to generate CoT tokens. Fix: pass `enable_thinking: false` in the request body when `prefill_no_think` is true.
